You are an expert Python security engineer. You are improving an existing modular 
web vulnerability scanner called BugBounty Hunter. The codebase structure is:

  bugbounty-hunter/
  ├── main.py
  ├── requirements.txt
  ├── modules/
  │   ├── recon.py        # Crawler + subdomain enumeration
  │   ├── scanner.py      # All active vulnerability checks
  │   ├── reporter.py     # HTML/JSON/TXT report generation
  │   └── utils.py        # Shared helpers, session factory, finding() dict

The finding() helper in utils.py returns:
  {"title": str, "url": str, "severity": str, "details": str, "evidence": str}

Severity levels: "critical", "high", "medium", "low", "info"

The session factory in utils.py creates a requests.Session pre-loaded with any
cookies/headers passed via CLI.

---

## OBJECTIVE

Refactor and extend the scanner so it:
1. Eliminates false positives caused by Single Page Application (SPA) shells
2. Validates authenticated sessions before scanning
3. Detects real, rewardable vulnerabilities on modern SaaS targets
4. Extracts and tests API/GraphQL surfaces
5. Detects IDOR via object reference manipulation
6. Mines JavaScript bundles for secrets and hidden endpoints

## PART 1 — modules/recon.py

### 1a. SPA Detection + Shell Fingerprinting

At crawler initialisation, fetch the root URL once and fingerprint the app shell:

  self.is_spa = False
  self.spa_shell_size = 0
  self.spa_shell_hash = ""

  def _fingerprint_shell(self):
      r = self.session.get(self.target, timeout=self.timeout)
      content = r.text.lower()
      spa_signals = [
          '<div id="root"', '<div id="app"', 'bundle.js',
          'chunk.js', '__next', 'ng-version', 'react', 'vue.js',
          'ember', 'backbone', '__webpack'
      ]
      score = sum(1 for s in spa_signals if s in content)
      if score >= 2:
          self.is_spa = True
          self.spa_shell_size = len(r.text)
          self.spa_shell_hash = hashlib.md5(r.text.encode()).hexdigest()

  def _is_real_page(self, response):
      if not self.is_spa:
          return True
      size_diff = abs(len(response.text) - self.spa_shell_size)
      content_hash = hashlib.md5(response.text.encode()).hexdigest()
      if content_hash == self.spa_shell_hash:
          return False   # identical shell — not a real page
      if size_diff < 512:
          return False   # near-identical — likely shell with minor variation
      return True

Every URL appended to self.discovered_urls must pass _is_real_page(). Discard
silently any URL whose response fails this check.

### 1b. Auth Validation

After _fingerprint_shell(), call:

  def _validate_auth(self):
      if not self.session.cookies and not self.session.headers.get("Authorization"):
          self.authenticated = False
          return
      # Make a request to a known auth-gated path and look for signals
      probe_paths = ["/api/v1/user", "/api/v1/me", "/graphql", "/api/graphql"]
      for path in probe_paths:
          try:
              r = self.session.get(self.target + path, timeout=self.timeout)
              if r.status_code in (200, 403) and r.status_code != 401:
                  self.authenticated = True
                  return
          except Exception:
              continue
      self.authenticated = False
      print("[!] WARNING: Session appears unauthenticated or expired.")

Store self.authenticated as a flag. Scanner modules can check it and skip or
adjust their behaviour accordingly.

### 1c. JavaScript Bundle Mining

After crawling, collect all .js URLs discovered. For each:

  def mine_js_bundles(self) -> list[dict]:
      findings = []
      secret_patterns = {
          "AWS Access Key":      r'AKIA[0-9A-Z]{16}',
          "Generic API Key":     r'["\']([a-zA-Z0-9_\-]{32,45})["\']',
          "Bearer Token":        r'(?i)bearer\s+([a-zA-Z0-9\-_.]{20,})',
          "Internal Endpoint":   r'["\']/(internal|admin|debug|private)/[^"\']{3,}["\']',
          "nr-data Endpoint":    r'https?://[a-z0-9\-]+\.nr-data\.net[^\s"\']*',
          "Hardcoded Password":  r'(?i)password\s*[:=]\s*["\']([^"\']{8,})["\']',
          "Private IP":          r'(?:10\.|172\.1[6-9]\.|172\.2[0-9]\.|192\.168\.)\d+\.\d+',
      }
      for js_url in self.js_urls:   # populate self.js_urls during crawl
          try:
              r = self.session.get(js_url, timeout=self.timeout)
          except Exception:
              continue
          for label, pattern in secret_patterns.items():
              matches = re.findall(pattern, r.text)
              for match in list(set(matches))[:5]:
                  findings.append(finding(
                      f"JS Secret: {label}", js_url, "high",
                      f"Pattern '{label}' matched in JS bundle",
                      match[:120]
                  ))
      return findings

During crawl, populate self.js_urls: any discovered URL ending in .js, or
any URL found inside <script src="..."> tags, goes into this list.

## PART 2 — modules/scanner.py

All existing scan methods must be patched to call self._confirm_finding(url,
evidence) before appending to findings. This secondary confirmation re-requests
the URL and checks the evidence string still appears in the response, preventing
ephemeral false positives.

  def _confirm_finding(self, url: str, evidence: str, method="GET",
                       data=None) -> bool:
      try:
          if method == "POST":
              r = self.session.post(url, data=data, timeout=self.timeout)
          else:
              r = self.session.get(url, timeout=self.timeout)
          return evidence in r.text
      except Exception:
          return False

### 2a. Patch XSS module

Current approach: inject payload into URL params, check if reflected.
Required improvements:

  - Use a diverse payload list covering HTML context, JS context, attribute
    context, and template injection:

      payloads = [
          '<svg/onload=alert(1)>',
          '"><img src=x onerror=alert(1)>',
          "';alert(1)//",
          '{{7*7}}',          # template injection canary
          '${7*7}',           # JS template literal
          '<script>alert(1)</script>',
          '"><script>alert(1)</script>',
          "javascript:alert(1)",
      ]

  - After reflection is detected, classify the context:
      * If payload is inside a <script> block → JS context XSS (higher severity)
      * If payload is in an attribute → Attribute XSS
      * If payload is in HTML body → Reflected XSS
      * If {{7*7}} reflects as "49" → Server-Side Template Injection (critical)

  - Call _confirm_finding() before appending. Only append confirmed findings.

### 2b. Patch SQLi module

Current approach: error-based + time-based.
Required improvements:

  - Add boolean-based blind detection. For each injectable param, send:
      * Original value → record response length L1
      * Value + " AND 1=1-- -" → record length L2  
      * Value + " AND 1=2-- -" → record length L3
      If L1 ≈ L2 and L1 ≠ L3 (differ by > 50 bytes), flag boolean-based SQLi.

  - For time-based, require the delay to occur on TWO separate requests before
    reporting (halves false positives from network jitter):

          self.session.get(url_with_sleep_payload, timeout=15)
          delays.append(time.time() - start)

  - Add the following additional error signatures to the error-based check:
      "ORA-", "PLS-", "DB2 SQL error", "SQLITE_ERROR",
      "Unclosed quotation mark", "quoted string not properly terminated",
      "mysql_fetch", "pg_query", "SQLSTATE"

### 2d. New Module — scan_graphql()

Add this full method to VulnScanner:

  def scan_graphql(self) -> list[dict]:
      findings = []
      endpoints = ["/graphql", "/api/graphql", "/nerdgraph/graphql",
                   "/v1/graphql", "/query"]
      headers = {"Content-Type": "application/json"}

      introspection = {"query": "{ __schema { types { name } } }"}
      typename      = {"query": "{ __typename }"}
      batch_payload = [{"query": "{ __typename }"}] * 50

      for ep in endpoints:
          url = self.base_url.rstrip("/") + ep

          # Test 1: Introspection
          try:
              r = self.session.post(url, json=introspection,
                                    headers=headers, timeout=self.timeout)
              if r.status_code == 200 and "__schema" in r.text:
                  if self._confirm_finding(url, "__schema", "POST",
                                           introspection):
                      findings.append(finding(
                          "GraphQL Introspection Enabled", url, "medium",
                          "Full schema is exposed via introspection. Attackers "
                          "can enumerate all types, queries, mutations, and "
                          "arguments.",
                          r.text[:400]
                      ))
          except Exception:
              continue

          # Test 2: Batching abuse
          try:
              r = self.session.post(url, json=batch_payload,
                                    headers=headers, timeout=self.timeout)
              if r.status_code == 200:
                  try:
                      resp_json = r.json()
                      if isinstance(resp_json, list) and len(resp_json) > 1:
                          findings.append(finding(
                              "GraphQL Query Batching Unrestricted", url,
                              "medium",
                              "Server accepts batched GraphQL arrays with no "
                              "apparent limit. This can be abused to bypass "
                              "rate limiting on mutations (e.g. brute-force "
                              "login, OTP enumeration).",
                              f"Sent 50 batched queries; received {len(resp_json)} responses"
                          ))
                  except Exception:
                      pass
          except Exception:
              pass

          # Test 3: Alias-based query amplification
          alias_query = {"query": " ".join(
              [f'q{i}: __typename' for i in range(100)]
          )}
          try:
              r = self.session.post(url, json=alias_query,
                                    headers=headers, timeout=self.timeout)
              if r.status_code == 200 and "q99" in r.text:
                  findings.append(finding(
                      "GraphQL Alias Amplification", url, "medium",
                      "Server does not limit query aliases. 100 aliased fields "
                      "returned successfully — potential request amplification "
                      "for DoS or rate-limit bypass.",
                      "q99 present in response"
                  ))
          except Exception:
              pass

      return findings

### 2e. New Module — scan_idor()

Add this full method to VulnScanner:

  def scan_idor(self) -> list[dict]:
      findings = []
      import re

      # Patterns to detect object references in URLs and response bodies
      id_patterns = [
          (r'[?&](account|accountId|account_id|user|userId|user_id|'
           r'org|orgId|org_id|id|guid|uuid|ref)=([0-9a-f\-]{4,36})', "param"),
          (r'/(accounts|users|orgs|organisations|entities)/([0-9a-f\-]{4,36})',
           "path"),
      ]

      candidates = []
      for url in self.discovered_urls:
          for pattern, ref_type in id_patterns:
              for m in re.finditer(pattern, url, re.IGNORECASE):
                  candidates.append({
                      "url": url,
                      "param": m.group(1),
                      "value": m.group(2),
                      "type": ref_type
                  })

      seen = set()
      for c in candidates:
          key = (c["param"], c["value"])
          if key in seen:
              continue
          seen.add(key)

          original_val = c["value"]
          original_url = c["url"]

          # Get baseline
          try:
              baseline = self.session.get(original_url, timeout=self.timeout)
          except Exception:
              continue

          # Attempt 1: Increment numeric IDs
          if original_val.isdigit():
              for delta in [-1, 1, -100, 100]:
                  test_val = str(int(original_val) + delta)
                  test_url = original_url.replace(original_val, test_val, 1)
                  try:
                      r = self.session.get(test_url, timeout=self.timeout)
                      if (r.status_code == 200
                              and len(r.text) > 500
                              and abs(len(r.text) - len(baseline.text)) < 5000
                              and r.text != baseline.text):
                          findings.append(finding(
                              "Potential IDOR — Numeric ID Manipulation",
                              test_url, "high",
                              f"Param '{c['param']}' changed from {original_val} "
                              f"to {test_val}. Server returned HTTP 200 with "
                              f"non-identical, non-empty content.",
                              f"Baseline size: {len(baseline.text)}, "
                              f"Test size: {len(r.text)}"
                          ))
                  except Exception:
                      continue

          # Attempt 2: Zero-out UUIDs (null reference test)
          uuid_pattern = r'^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$'
          if re.match(uuid_pattern, original_val, re.IGNORECASE):
              null_uuid = "00000000-0000-0000-0000-000000000000"
              test_url  = original_url.replace(original_val, null_uuid, 1)
              try:
                  r = self.session.get(test_url, timeout=self.timeout)
                  if r.status_code == 200 and len(r.text) > 500:
                      findings.append(finding(
                          "Potential IDOR — Null UUID Accepted",
                          test_url, "medium",
                          f"Replacing UUID in '{c['param']}' with null UUID "
                          f"returned HTTP 200 with content.",
                          f"Response size: {len(r.text)} bytes"
                      ))
              except Exception:
                  pass

      return findings
